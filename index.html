O que é Pair Programming?

Dois desenvolvedores, um código — e agora, uma IA

Pair Programming Tradicional

Dois devs trabalham juntos: o piloto escreve o código e o navegador revisa, sugere e pensa estrategicamente. Os papéis se alternam constantemente.

Programação Assistida com IA

O navegador humano é substituído por uma IA. Você continua sendo o piloto — a IA é seu copiloto, revisando, gerando e depurando código em tempo real.

Gera código

Cria funções e estruturas completas a partir de uma descrição

Refatora

Melhora legibilidade e boas práticas sem alterar o comportamento

Depura

Identifica, explica e corrige erros no código

Documenta

Explica código linha a linha e gera documentação automaticamente

Ferramentas

As principais ferramentas de programação assistida

GitHub Copilot

Integrado ao VS Code. Sugere código em tempo real, gera funções, explica, corrige e interage via chat. O modelo raciocina e usa ferramentas para agir no seu ambiente local.

VS Code Local Chat
Replit

Ambiente de desenvolvimento online completo com IA integrada. Permite criar, executar e publicar projetos diretamente no navegador, sem instalação.

Como a IA raciocina — exemplo
// 1. Entender o problema
"O usuário quer verificar se um número é par. Vou gerar código Python."

// 2. Escolher a ferramenta
"Para escrever em um arquivo, vou usar `edit`."

// 3. Tool call
{ "tool": "edit", "file": "main.py",
  "content": "n = 7\nif n % 2 == 0:\n print('Par')\nelse:\n print('Ímpar')" }

// 4. Sistema executa → resultado volta ao modelo

Práticas Realizadas

Quatro práticas utilizando GitHub Copilot no VS Code

Prática 1
IA como Copiloto

Geração de uma função Python para verificar se um número é primo, com explicação linha a linha, documentação em Markdown e versão otimizada aplicando Clean Code. Complexidade reduzida de O(n) para O(√n).

num_primos.py explicacao_num_primo.md
Prática 2
Refatoração

Refatoração de código mal estruturado: nomes genéricos (c, l, t) renomeados, range(len()) substituído por funções nativas, retorno em tupla transformado em @dataclass e docstrings adicionadas.

refatoracao.py explicacao-refatoracao.md
Prática 3
Debug com IA

Identificação e correção de 4 erros: SyntaxError (string sem aspas), TypeError (input() sem conversão numérica), erro lógico (f-string sem prefixo f) e IndentationError (bloco if sem indentação).

debug.py explicacao-debug.md
Prática 4
IA Explicando Código

O código refatorado da Prática 2 foi submetido à IA com o prompt "Explique este código linha por linha", validando a capacidade interpretativa do modelo e a qualidade da refatoração realizada.

explicacao-refatoracao.md

Boas Práticas

Como usar IA no desenvolvimento de forma responsável

Faça
  • Validar o código gerado antes de usar
  • Testar sempre em ambiente controlado
  • Pedir explicação para entender o que foi gerado
  • Refinar o prompt até obter o resultado esperado
  • Usar a IA como apoio, não como resposta final
Evite
  • Copiar código sem entender o que ele faz
  • Confiar cegamente na saída da IA
  • Usar código sem testar
  • Depender totalmente da IA para decisões técnicas

Se você não entende o código,
você não deve usar esse código.
